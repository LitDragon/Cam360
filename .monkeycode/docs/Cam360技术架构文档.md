# Cam360技术架构文档

- 文档版本：v4.0
- 更新时间：2026-04-22
- 文档状态：当前约束版
- 适用范围：Cam360 当前仓库

## 1. 目标

本文件只保留当前仍有效的架构约束、模块边界和阶段性路线，不再重复维护细碎实现说明。

## 2. 当前架构事实

- 当前阶段仍是 `M0 骨架`。
- App 入口为 UIKit 生命周期桥接 + SwiftUI 根视图。
- 根级结构由 `AppBootstrap`、`AppRouter`、`AppContainer`、`AppRootView` 驱动。
- 当前主界面只有 3 个 tab：`dashboard`、`gallery`、`settings`。
- `DeviceOnboarding` 和 `Settings` 已有可用骨架。
- `LivePreview`、`Playback`、`Downloads` 仍是占位实现。
- `DeviceSession` 已有本地状态机骨架，但没有接入真实设备通信。
- 当前测试 target 只有 `Cam360Tests`。

## 3. 技术基线

- 开发语言：Swift
- 最低支持版本：`iOS 13`
- 主路径实现：优先沿用 `iOS 15` 时代稳定写法
- `iOS 17+` 能力必须放在 `#available` 分支内
- UI：SwiftUI 为主，UIKit 生命周期桥接
- 状态模型：`ObservableObject`、`@Published`、`@ObservedObject`、`@State`
- 持久化：`UserDefaults` + App Sandbox
- 第三方依赖：当前已见 `Lottie`

## 4. 目录与边界

- `App`：生命周期、根路由、依赖装配、根视图
- `Feature`：页面渲染、用户动作、页面状态
- `Core`：跨 Feature 复用的稳定薄能力

约束：

- 按 Feature 拆目录，不回退到纯 `Views/`、`ViewModels/` 平铺。
- Feature 不直接持有长生命周期连接、播放器或下载任务控制权。
- 共享依赖统一从 `AppContainer` 组合后下发。
- 不引入重型 Clean Architecture 或第三方 DI 框架。

## 5. 导航与依赖注入

- 首次安装或未完成 onboarding：进入 `DeviceOnboarding`
- 已完成 onboarding：进入主界面
- `AppContainer` 当前组合：
  - `KnownDeviceRepository`
  - `AppPreferenceStore`
  - 各 Feature Store

## 6. M1+ 路线

按以下顺序逐步接入真实能力：

1. AP onboarding 真实接入
2. `DeviceSession` 作为统一状态源
3. 实时预览
4. 回放
5. 下载导出
6. 真实设备设置读写

## 7. 文档与验证约束

- 文档默认必须精简，避免在多个文件里重复展开同一事实。
- 长期事实写根目录 `PROJECT_CONTEXT.md`，短期计划写根目录 `TASKS.md`。
- 文档和代码冲突时，以已检查代码为准。
- 本地默认不做手动模拟器验证；CI 可继续跑基于 iOS Simulator 的 build/test。
